## Diagnostic context (added by Claude Code, 2026-04-20)

### Repo source ≠ Live site (HIGH severity)
The HTML served at `https://skipschool.ai/<slug>` does not match `site/files/src/<slug>.html`.
Example — `05-ai-curriculum-builder` footer column:

- Source repo: `<h4>About</h4><a href="/about">Our Story</a><a href="/contact">Contact</a>`
- Live HTML: `<div ...><strong>About</strong></div><a href="/about-ashley-larkin">Our Story</a><a href="/cdn-cgi/l/email-protection#…">Contact</a>`

The live build is from a different source than what's currently committed at `site/files/src/`.
Any "fix" applied to the repo will not land on the live site until the deploy pipeline is reconciled.

### Cloudflare Email Address Obfuscation is on, and broken (explains 100+ "broken links")
`server: cloudflare` is in the response headers — skipschool.ai is proxied through
Cloudflare in front of Vercel. Cloudflare's Email Address Obfuscation feature rewrites
mailto / contact links to `/cdn-cgi/l/email-protection#HEX`. Those URLs currently return
HTTP 404 even though they should be served by Cloudflare's CDN worker.

That single misconfiguration is the source of ~100 of the 114 broken-internal-link entries.
Real fix: turn off Email Address Obfuscation in Cloudflare dashboard
(Scrape Shield → Email Address Obfuscation → Off), or fix the underlying CF service.

scripts in `deploy/`), not as a Next.js layout component.
